原因是 v4.3 GAS 雖然已有郵筒、聊天室狀態與通話，但漏了 createInvite / acceptInvite 路由。

- 儲存後：部署 → 管理部署作業 → 編輯 → 新版本 → 部署。

v4.4 新增／修正
--------------
- createInvite：建立一次性邀請與雙方獨立頻道。
- acceptInvite：對方接受後立即刪除邀請，不能被第二次使用。
- inviteStatus：邀請者可看到等待中／已加入／失效／已取消。
- cancelInvite：邀請者可主動取消尚未使用的邀請。
- 邀請後端不保存名稱、訊息或 #angkey；解密鑰匙仍只存在完整邀請網址的 # 後面。
- API 18 秒逾時保護，GAS 沒重新部署時不會一直卡在「建立中／加入中」。
- 前端舊限制同步為：未讀最長 6 小時、進房後未開 5 分鐘、點開 15～60 秒。

其餘沿用 v4.3
------------
- 鍵盤展開後固定滿版。
- 以是否真的在聊天室判斷紅綠燈與新未讀。
- 未讀水滴逐則點開。
- 進聊天室後該批訊息最多 5 分鐘。
- 點開後 15／20／30／60 秒消失，雙方同步。
- 敲一下、同步離開、整段對話結束。
- 語音通話接通後縮成上方小列，可在私・話內邊講邊傳訊息。

更新後測試
----------
1. 先重新部署 GAS 新版本。
2. 清除網站快取，或移除舊的主畫面 App 後重新加入。
3. A 按「建立專屬對話」，應立即出現邀請網址與黃色「等待對方加入」。
4. B 用完整網址開啟，網址必須包含 ?invite= 與 #angkey=。
5. B 接受後，A 的邀請狀態應變綠色「對方已加入」。
6. 同一邀請再開第二次，應顯示已使用或已失效，不會建立第二個人。

本版完成項目
------------
- 鍵盤展開後仍固定滿版，只有中間訊息區可滑動。
- 對方狀態燈：在聊天室／暫時離開／已離開。
- 判定以「訊息送達時，對方是否真的在聊天室內」為準。
- 對方不在聊天室時收到的訊息，維持長時間加密未讀。
- 進入聊天室後，當下那批未讀開始 5 分鐘閱讀時段。
- 離開聊天室後新收到的訊息，重新算成新未讀，不併入上一輪。
- 未讀改為側邊水滴，一顆水滴對應一則，可逐則點開。
- 點開內容後依房間同步設定 15／20／30／60 秒消失。
- 房間時間由任一方修改後同步給另一方。
- 「敲一下／叮」不建立聊天訊息，只提醒對方注意。
- 一方按「結束對話」，雙方都退出並顯示「這段對話已結束」。
- 語音通話接通後可縮成上方小列，聊天室內仍能打字。
- 通話支援接聽、拒絕、靜音、聲音開關與掛斷。

重要說明
--------
1. 未進聊天室的密封包預設保留 6 小時。這是 GAS CacheService 單筆快取可設定的上限。
2. 水滴被點開後，伺服器上的密封包立即刪除；解密後內容只在當前頁面短暫顯示。
3. 目前「通知」在網頁仍執行時可用。若 iPhone／iPad 已完全關閉 PWA，真正的背景推播仍需另外接 Web Push 訂閱與推播伺服器。
4. 通話目前使用 WebRTC STUN 測試；部分公司網路、嚴格 NAT 或電信網路可能需要 TURN 才能穩定連線。
5. Cloudflare Worker／Pages Function 若本來已經把任意 /api/<action> 轉送 GAS，這次不需要修改。

更新後測試
----------
- 建議先清除網站快取，或把已加入主畫面的舊版移除後重新加入。
- 用兩支不同手機建立／接受同一個邀請。
- A 留在聊天室，B 傳訊息：A 應直接看到新水滴並進入目前閱讀輪次。
- A 離開聊天室，B 再傳訊息：該訊息應保留為新的未讀水滴。
- 點水滴後測試 20 秒消失；再修改成 15 秒，確認另一台同步更新。
- 測試「叮」、狀態燈、結束對話與語音通話縮小列。
